### 2. P1（float vs int）根因分析 ✅ 已修复
**从 C# 源码确认根因：**
- 读了 `ServerpartHelper.cs` 的 `BindDataRowToModel` 方法
- 读了 `RTSERVERPARTSHOPModel.cs` 的类型声明
- **发现**：C# 用 `TryParseToDouble()` → `System.Double?` → JSON 保留小数点（如 `5518.0`）
- **之前的错误**：Python `database.py` 中把 Decimal/float 转成了 int（方向反了！）
- **修复**：`database.py` 第273-282行，Decimal → `float(v)` only，不再转 int

### 3. P3（日期格式）根因分析 ✅ 已修复
**从 C# 源码确认根因：**
- 读了 `RTSERVERPARTSHOPHelper.cs` 的 `BindDataRowToModel`
- **发现**：C# Model 日期字段多为 `string` 类型，用 `drXXX["col"].ToString()` 赋值
- Windows zh-CN 环境 `DateTime.ToString()` 输出 `yyyy/M/d H:mm:ss`（月/日/时不补零，分/秒补零）
- Python 用 `strftime('%Y-%m-%dT%H:%M:%S')` 输出 ISO 格式（有 T 分隔符，全补零）
- **修复**：`database.py` 中 datetime → `f"{v.year}/{v.month}/{v.day} {v.hour}:{v.minute:02d}:{v.second:02d}"`
- date → `f"{v.year}/{v.month}/{v.day} 0:00:00"`
