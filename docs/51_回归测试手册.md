# 回归测试手册

## 快速开始

```powershell
# 冒烟测试（<1分钟，20个核心接口）
python scripts/regression.py smoke

# 标准测试（~3分钟，329个端点响应码）
python scripts/regression.py standard

# 全量对比（~15分钟，需 C# API 在线）
python scripts/regression.py full
```

## 三档测试说明

| 档位 | 范围 | 耗时 | 是否需原 API | 触发时机 |
|------|------|------|-------------|---------|
| smoke | 20 个核心接口 | <1分钟 | 否 | 每次改代码后 |
| standard | 329 个 EShangApiMain 端点 | ~3分钟 | 否 | 每次提交前 |
| full | 逐字段数据对比 | ~15分钟 | 是 | 每轮迭代后 |

## 判定标准

### 冒烟测试（smoke）
- HTTP 200 且 `Result_Code` 为 100 或 200 即 PASS
- 系统端点（`/health`、`/`）200 即 PASS

### 标准测试（standard）
- HTTP 200 即 PASS
- 超时（10秒）标记为 TIMEOUT 不计入 FAIL

### 全量对比（full）
- 复用 `compare_api.py` 的完整对比逻辑
- 需要 `endpoint_case_library.json` manifest

## 测试结果

结果自动保存到 `scripts/test_results/`：
- `smoke_latest.json` — 冒烟测试结果
- `standard_latest.json` — 标准测试结果
- `full_latest.md` — 全量对比报告

## 新增接口后

1. 更新 `regression.py` 中的 `SMOKE_ENDPOINTS`（如果是核心接口）
2. 更新 `endpoint_case_library.json`（如果是 EShangApiMain 接口）
3. 运行 `python scripts/regression.py smoke` 验证
