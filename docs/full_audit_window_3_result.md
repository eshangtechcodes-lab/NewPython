## 模块整改建议

### Audit
- 先回退 `GetSpecialBehaviorReport`、`GetAbnormalRateReport` 的“已迁移完成”状态；空列表实现不能上线。
- `GetAuditDetils`、`UpLoadAuditExplain`、`IssueAuditTasks` 必须按原 helper 全量迁回，不能再通过 `AUDITTASKS/ABNORMALAUDIT` 通用 CRUD 兜底。
- 对所有 Audit 路由清理 batch router 的统一 `pk_val` / 小写包装，恢复原 C# JsonMsg 契约。

### Analysis
- 先补齐 4 条 `SPCONTRIBUTION` 缺失路由，再处理 11 组 CRUD 壳化问题。
- `ReCalcCACost`、`GetRevenueEstimateList`、`Solid*`、`Get*Report` 这类分析/固化接口必须优先从 helper 级迁回，风险高于普通 CRUD。
- 对 `ASSETSPROFITS`、`PROFITCONTRIBUTE`、`PERIODMONTHPROFIT` 的树形/固化/报表逻辑做专项回归，不可再按单表分页视为完成。

### BusinessMan
- 先修正表映射：`BusinessMan/BusinessManDetail/Commodity` 当前分别漂移到 `T_BUSINESSMAN/T_BUSINESSMANDETAIL/T_COMMODITY`，与原 OWNERUNIT/OWNERUNITDETAIL/COMMODITY_BUSINESS 不一致。
- `CreateBusinessMan`、`AuthorizeQualification`、`GetUserList` 不能用 CRUD 模板迁移，必须按原 helper 恢复关联写入、授权历史和用户树。
- `GetUserList` 还要恢复 GET/POST 双入口与原查询参数。

### Supplier
- 先下线 `Supplier/DeleteQUALIFICATION_HIS`，这是 Python 额外暴露的删除入口。
- `GetSupplierTreeList`、`RelateBusinessCommodity`、`SynchroQualification` 要按 helper 迁回树形和关联逻辑，不能继续以空返回或通用 upsert 代替。
- `GetSupplierList` 需恢复资质数量/OtherData 等原返回内容，否则前端列表契约不完整。

### Verification
- `VerifyEndaccount`、`ApproveEndaccount`、`SubmitEndaccountState`、`ApplyEndaccountInvalid`、`CancelEndaccount` 属于核心状态机，必须优先按 EndaccountHelper 全量回迁。
- 所有 `return []`、`return {}`、`return True, ""`、`return {"status": "ok"}` 的接口都应直接判定为未迁移完成。
- `GetEndAccountData`、`GetCommoditySaleList`、`GetMobilePayDataList` 这类查询接口不能继续复用日结主表或空实现，需要按 DataVerificationHelper 的数据类型分支恢复。

### Sales
- `DeleteCOMMODITYSALE` 要先确认是否保持原“不可删/空实现”语义；当前 Python 已经产生真实删除行为，风险为 P0。
- `GetCommoditySaleSummary`、`GetCommodityTypeSummary`、`GetCommodityTypeHistory`、`SaleRank` 必须恢复统计接口，不得落成 `T_COMMODITYSALE` 原表分页。
- `RecordSaleData`、`UpdateEndaccountError`、`UpdateCommoditySale` 属于业务写接口，应按原 helper/流程重建，不可继续使用通用 `_synchro`。

## 风险与阻塞

- 本次为静态一致性审计，未连接数据库执行动态 SQL；但多数问题在 controller/service/helper 签名和实现层已经能直接判定，不影响“不一致”结论。
- 多个 Python 路由由 batch router 循环生成，同一个模板缺陷会批量扩散；因此 Analysis、BusinessMan、Supplier 的问题具有系统性。
- 部分 C# helper 还会继续下探到 AutoBuild/SQL 拼接层；本次只在能静态追到 helper 时给证据，已足够证明 Python 现状不是等价迁移。
- 文档仅更新 `docs/full_audit_window_3_result.md`，未改动业务代码和 baseline JSON。
