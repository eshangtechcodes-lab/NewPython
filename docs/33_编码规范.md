# 编码规范

> 从 16 个迁移踩坑 + SOP 历史 Bug 中提炼的编码规范。每次提交前自查。

## 1. 代码分层

| 层 | 职责 | 示例 |
|----|------|------|
| Router | HTTP 协议处理、参数校验、响应封装 | 路由装饰器、`Depends(get_db)`、`Result.success()` |
| Service | 业务逻辑、SQL 查询、数据转换 | `get_xxx_list()`、`_build_where_sql()` |
| Model | 数据结构定义 | `SearchModel`、`Result`、`JsonListData` |
| Core | 基础设施 | `DatabaseHelper`、`config`、中间件 |

**规则**：Router 不写 SQL，Service 不引用 `Request`/`Response` 对象。

## 2. SQL 规范

### 2.1 达梦兼容
- **禁止** `LIMIT OFFSET` — 用 `ROWNUM` 子查询分页
- **禁止** `(+)` 外连接 — 用 `LEFT JOIN`
- **注意** `CONNECT BY` 树形查询在达梦中需测试兼容性

### 2.2 参数安全
- 用户输入参数**必须**检查类型和范围
- 整数参数用 `int()` 强转后拼接
- 字符串参数用单引号包裹并转义 `'` → `''`
- **禁止**直接拼接未经处理的用户输入到 SQL

### 2.3 SQL 来源
- Service 中的 SQL **必须参考原 C# Helper**，不可自创
- 复杂 SQL 需在注释中标注原 Helper 方法名

## 3. 空值处理

| 场景 | 错误写法 | 正确写法 |
|------|---------|---------|
| 检查空字符串 | `if not val:` | `if val is None or val == "":` |
| 字符串字段 null 转空 | 不处理 | `row[f] = "" if row.get(f) is None` |
| 数值字段 null | `str(val)` → `"None"` | `val if val is not None else 0` |
| 日期字段 null | 不处理 | 显式返回 `None` |

**关键**：`if not val` 会把 `0`、`""` 都当作 False，在业务逻辑中几乎总是错误的。

## 4. 日期格式

C# 风格日期格式：`2023/3/30 14:02:47`（月日不补零）

```python
# 标准转换函数
def _format_date(val):
    if val is None:
        return None
    dt = datetime.strptime(str(val)[:19], "%Y-%m-%d %H:%M:%S")
    return f"{dt.year}/{dt.month}/{dt.day} {dt.hour}:{dt.minute:02d}:{dt.second:02d}"
```

## 5. 路由规范

- 路由路径**必须与原 C# `[Route()]` 注解完全一致**（大小写敏感）
- Delete 接口用 `@router.api_route(methods=["GET", "POST"])`（C# 同时支持 GET/POST）
- 参数名的 PascalCase 与原 C# 保持一致（如 `XXXId` 不是 `xxx_id`）

## 6. Service 必备组件

每个 Service 文件头部必须定义：

```python
TABLE_NAME = "T_XXX"              # 主表名
PRIMARY_KEY = "XXX_ID"            # 主键
EXCLUDE_FIELDS = {"xxx", "yyy"}   # 排除的查询参数
DATE_FIELDS = {"OPERATE_DATE"}    # 日期字段
EXTRA_FIELDS = {"XXX": None}      # C# Model 回显属性
STRING_FIELDS = {"XXX_NAME"}      # 字符串字段（null→""）
```

## 7. 已知陷阱速查

| ID | 陷阱 | 自查方法 |
|----|------|---------|
| P1 | 达梦 `NUMBER` 迁移为 `VARCHAR` | `ALTER TABLE MODIFY col INT` |
| P3 | 默认分页缺失导致返回全量 | 无分页时 `rows[:10]` |
| P4 | 路由大小写错误返回 404 | 对照原 `[Route()]` 注解 |
| P10 | Python 3.8 类型注解 | 文件头 `from __future__ import annotations` |
| P11 | 达梦无序列权限 | `MAX(ID)+1` 降级 |
| P14 | 软删除 vs 真删除混用 | 查原 Helper Delete 方法体 |
| P16 | Header 参数遗漏 | 查原 C# `GetIntHeader`/`GetStringHeader` |
| P17 | 前端非数据库字段 | `EXCLUDE_FIELDS` 过滤 |
| SOP-1 | `_build_where_sql` 变量未定义 | 搜索文件确认无未定义变量 |
| SOP-2 | 漏同步关联表 | `grep FROM service.py` 检查所有表 |

## 9. CommercialApi Service 规范

> 与 EShangApiMain Service 的区别：CommercialApi 以只读分析查询为主，不涉及 CRUD 操作。

### 9.1 文件组织

```
services/commercial/
├── service_utils.py          # 公共工具（safe_float/safe_int/get_province_id/date_no_pad 等）
├── revenue_*_service.py      # 营收相关（14 个文件）
├── bigdata_*_service.py      # 大数据相关（4 个文件）
└── {module}_service.py       # 其他模块（6 个文件）
```

### 9.2 导入规范

```python
# 必须使用 service_utils 的公共函数，禁止本地重复定义
from services.commercial.service_utils import (
    safe_float as _sf, safe_int as _si,
    get_province_id, build_sp_where,
    date_no_pad,  # 日期不补零格式化
)
```

### 9.3 函数命名

| C# 路由 | Python 函数 | 说明 |
|---------|------------|------|
| `GetRevenuePushList` | `get_revenue_push_list()` | PascalCase → snake_case |
| `GetServerpartBrand` | `get_serverpart_brand()` | 保持业务语义 |

### 9.4 与 EShangApiMain Service 的差异

| 维度 | CommercialApi Service | EShangApiMain Service |
|------|----------------------|----------------------|
| **常量定义** | 不需要 TABLE_NAME 等 | 必须定义 6 个常量 |
| **操作类型** | 只读查询为主 | 增删改查 |
| **SQL 来源** | 原 C# AccountHelper | 原 C# Controller |
| **分页** | 一般不分页 | 使用 SearchModel 分页 |
