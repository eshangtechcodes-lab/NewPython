### 3.3 关键业务逻辑

#### GetBrandList（核心，L40-210）

1. 基础查询 `SELECT * FROM T_BRAND` + 通用 WHERE
2. **BRAND_INDUSTRY 子类型递归**：通过 `T_AUTOSTATISTICS` 表递归查找所有子级业态 ID
3. **SPREGIONTYPE_IDS / SERVERPART_IDS 过滤**：通过 `T_SERVERPART JOIN T_SERVERPARTSHOP` 的 `WM_CONCAT(DISTINCT BUSINESS_BRAND)` 筛选关联品牌
4. **BRAND_CATEGORY=2000 分支**：商城品牌走 SQL 级分页；其他品牌走 Python 内存分页
5. **关联查询**（查完品牌后额外执行 3 个 SQL）：
   - `T_AUTOSTATISTICS` → 解析 BUSINESSTRADE_NAME（经营业态名称）
   - `T_COOPMERCHANTS + T_RTCOOPMERCHANTS` → MerchantID / MerchantName
   - `T_SERVERPARTSHOP` → ServerpartList（关联服务区列表，`[{label, value}]`）
6. **最终排序**：按 `ServerpartList.Count` 降序

#### GetCombineBrandList（极复杂，L224-738）

- 按 `WECHATAPPSIGN_ID` 分组品牌
- 聚合 BRAND_INDUSTRY / BRAND_TYPE
- 关联商户数据和服务区门店数据
- 原 C# 还涉及 CONTRACT_STORAGE / PLATFORM_DASHBOARD 的营收分润计算（当前 Python 简化处理）

#### GetTradeBrandTree（递归树，L1113-1236）

- 查询 `T_AUTOSTATISTICS`（经营业态，AUTOSTATISTICS_TYPE=2000）
- 查询 `T_BRAND`
- 递归 `BinData()` 方法按 `AUTOSTATISTICS_PID` 构建嵌套树
- 每个业态节点下挂载 `BrandTreeList`（该业态下的品牌列表）

#### SynchroBrand（L944-1006）

- 排除字段（不入库）：`BUSINESSTRADE_NAME, SPREGIONTYPE_IDS, SERVERPART_IDS, MerchantID, MerchantID_Encrypt, MerchantName, ServerpartList`
- 验证 BRAND_INDUSTRY 必须为子级（无子类型才允许保存）
- 新增：序列 `SEQ_BRAND.NEXTVAL` 或降级 `MAX(ID)+1`
- 更新：按 BRAND_ID WHERE 条件

#### DeleteBrand（L1010-1026）

- 软删除：`UPDATE T_BRAND SET BRAND_STATE = 0 WHERE BRAND_ID = ?`

## 四、迁移脚本优化（两轮）

### 4.1 第一轮：性能优化

**文件**：`scripts/server_migrate.py`

| 改动 | Before | After |
|------|--------|-------|
| 写入方式 | 逐行 `execute()` | `executemany()` 批量写入 |
| 批量大小 | 500 | 2000 |
| 校验连接 | 复用旧连接 | 重建连接（避免 ORA-03113） |

**实测效果**：T_SERVERPARTSHOP 7078 条 30 分钟 → **4.3 秒**（提速 420x）

### 4.2 第二轮：自动同步改造

| 改动 | Before | After |
|------|--------|-------|
| 行为控制 | `MERGE_MODE=True/False` 手动切换 | **智能模式**（自动判断） |
| 表不存在 | CREATE + INSERT | CREATE + INSERT（不变） |
| 表存在 | **跳过**（MERGE_MODE=False 时） | **MERGE INTO 自动合并差异** |

**遇到的问题**：
- P18：T_AUTOSTATISTICS 旧表（非脚本建的）结构与 Oracle 不一致，MERGE 报错 `无效的列名`
- 解决：DROP 旧表 → 重新运行脚本（自动按 Oracle 最新结构建表）
