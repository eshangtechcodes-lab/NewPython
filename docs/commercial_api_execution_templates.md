## 6. Controller 验收清单模板

按 Controller 完成首轮内容验收时，建议至少核对以下项目。

```md
## Controller: xxx

- [ ] 路由与请求方式确认
- [ ] 参数定义确认
- [ ] 空参样本准备
- [ ] 文档样本准备
- [ ] 真实数据样本准备
- [ ] 返回结构对比
- [ ] 字段类型对比
- [ ] 默认值对比
- [ ] 排序对比
- [ ] 分页对比
- [ ] 日期格式对比
- [ ] 数值精度对比
- [ ] 异常返回对比
- [ ] 数据依赖表确认
- [ ] 差异已归因
- [ ] 修复后已回归
- [ ] 已沉淀技术要点
- [ ] 已沉淀业务规则
- [ ] 已沉淀踩坑记录
```

## 8. 推荐使用顺序

1. 先用 Controller 验收清单开任务
2. 执行过程中每日填写工作日志
3. 发现问题时建立接口问题单
4. 形成稳定结论后写入 AI Context
5. 可复用坑点补入踩坑文档
