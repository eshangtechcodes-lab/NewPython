# 部署运维手册

## 1. 环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.11+ | 运行 FastAPI 应用 |
| 达梦数据库 | DM8 | 192.168.1.99:5236 |
| Redis | 5.0+ | 缓存（可选） |
| Docker | 20.10+ | 容器部署（可选） |

## 2. 快速部署

### 方式 A：Docker（推荐）

```powershell
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env 填入实际密码

# 2. 构建并启动
docker-compose up -d

# 3. 查看状态
docker-compose ps
docker-compose logs -f api

# 4. 健康检查
curl http://localhost:8080/health
```

### 方式 B：直接运行

```powershell
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境变量（或创建 .env 文件）
cp .env.example .env

# 3. 启动（开发模式）
python main.py

# 4. 启动（生产模式）
uvicorn main:app --host 0.0.0.0 --port 8080 --workers 4
```

## 3. 配置说明

所有配置通过环境变量或 `.env` 文件设置，参见 `.env.example`。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DEBUG` | True | 调试模式 |
| `DM_HOST` | 192.168.1.99 | 达梦主机 |
| `DM_PORT` | 5236 | 达梦端口 |
| `DM_USER` | NEWPYTHON | 数据库用户 |
| `DM_PASSWORD` | — | 数据库密码 |
| `REDIS_HOST` | 127.0.0.1 | Redis 主机 |
| `LOG_LEVEL` | INFO | 日志级别 |

## 4. 运维操作

### 日志查看

```powershell
# 查看最新日志
Get-ChildItem logs/ | Sort-Object LastWriteTime -Descending | Select-Object -First 3

# 查看今天的日志
Get-Content logs/$(Get-Date -Format 'yyyyMMdd').log -Tail 50

# Docker 日志
docker-compose logs --tail 100 api
```

### 健康检查

```powershell
# 检查 API 和数据库连接
Invoke-RestMethod http://localhost:8080/health
# 返回: Result_Code=100 且 database=connected 表示正常
```

### 重启服务

```powershell
# Docker
docker-compose restart api

# 直接运行
Get-Process -Name python | Stop-Process -Force
Start-Sleep 2
python main.py
```

### 回归测试

```powershell
# 每次部署后运行冒烟测试
python scripts/regression.py smoke

# 完整测试
python scripts/regression.py standard
```

## 6. 故障排查

| 症状 | 排查步骤 |
|------|---------|
| 启动失败 | 检查 `.env` 配置 → 检查 `pip list` 依赖 → 查看终端错误 |
| /health 返回 999 | 数据库不可达 → `ping 192.168.1.99` → 检查 DM 服务状态 |
| 接口 404 | 检查 `main.py` 路由注册 → 路径大小写 |
| 接口 500 | 查看 `logs/` 日志 → 对照 `docs/33_编码规范.md` 踩坑清单 |
| 数据不一致 | 运行 `python scripts/regression.py full` 全量对比 |
