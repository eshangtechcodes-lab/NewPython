# 工作日志 Workflow

## 触发时机
- 会话结束前
- 用户主动要求保存日志
- 完成较大功能模块后

## 日志格式

文件路径：`workLog/YYYY-MM-DD.md`

```markdown
# YYYY-MM-DD 工作日志 — [一句话标题]

## 📋 工作内容
- 逐项列出今天完成的工作
- 每项包含：目标、修改方式、结果

## 📂 相关材料
- 涉及的文件、分支、依赖、文档

## 🔧 修改内容
- 具体的文件修改和行数变化
- Git commit 列表

## ⚠️ 踩坑记录
- 遇到的问题和解决方案
- 需要注意的技术细节

## 📊 验证结果
- 测试通过率
- 关键指标

## 📝 总结
- 本次工作的整体评价
- 学到的经验教训

## 🔜 下一步工作
- [ ] 待办事项 1
- [ ] 待办事项 2
```

## 操作步骤

// turbo-all

1. 检查 `git status`，确保所有改动已提交
2. 收集本次会话中所有 commit 信息
3. 按模板格式生成工作日志
4. 保存到 `workLog/YYYY-MM-DD.md`（如已存在则追加或覆盖）
5. `git add workLog/ && git commit -m "chore: 保存工作日志 YYYY-MM-DD"`
