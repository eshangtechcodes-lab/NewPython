# 文档维护工作流

> 当代码发生变更（新增接口、新增脚本等）后，自动重新生成相关文档保持同步。

// turbo-all

## 触发时机

以下情况需要执行此工作流：
1. 新增或删除了路由接口
2. 新增或删除了 scripts 目录下的脚本
3. 修改了中间件参数映射
4. 用户要求更新文档

---

## 步骤 1：重新生成接口全量清单

运行扫描脚本重新生成 `docs/21_接口全量清单.md`：

```powershell
python /tmp/gen_docs.py
```

如果脚本不存在，可以用以下方式快速生成：

```python
# 核心逻辑：扫描 main.py 的 include_router 和各路由文件的 @router.get/post 装饰器
# 输出为 docs/21_接口全量清单.md
```

---

## 步骤 2：验证文档

1. 打开生成的文件确认格式正确
2. 检查端点总数是否与预期一致
3. 如果有已知的迁移状态，确认状态标注正确

---

## 步骤 3：提交更新

```powershell
git add docs/
git commit -m "docs: 更新知识文档"
```

---

## 文档编号规范

| 编号段 | 分类 | 说明 |
|--------|------|------|
| 1x | 项目架构 | 项目总体设计和架构文档 |
| 2x | 接口文档 | API 清单、接口规格说明 |
| 3x | 开发知识 | 迁移经验、中间件映射、编码规范 |
| 4x | 工具资源 | 脚本清单、运维手册 |
| 5x | 质量保障 | 测试手册、性能基线 |

新增文档按编号顺序递增，例如：
- `12_数据库字典.md`
- `33_编码规范.md`
- `51_回归测试手册.md`
