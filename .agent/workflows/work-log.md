# 工作日志保存工作流

## 触发时机
每次会话**结束前**或用户说"结束"/"收工"/"今天到这"等类似话语时，自动执行此工作流。

## 步骤

### 1. 收集今日信息
// turbo
- 运行 `git log --format="%h %s" --after="$(Get-Date -Format 'yyyy-MM-dd')T00:00:00+08:00"` 获取今日提交列表
// turbo
- 运行 `python /tmp/show_stats.py` 获取最新PASS/DIFF统计（如果脚本存在）

### 2. 生成日志内容
按以下模板生成日志，保持与 `workLog/` 目录中已有日志的风格一致：

```markdown
# YYYY-MM-DD 工作日志

## 今日完成

### 1. [任务标题] ✅
- 具体工作内容
- 修复了什么问题
- 关键技术细节

## 当前测试状态
- **PASS: X/Y (Z%)**
- **DIFF: N个**

### 差异变化明细（如有）
| 接口 | 修改前 | 修改后 | 变化 |
|---|---|---|---|

## 关键发现（如有）
- 重要的技术发现或阻塞问题

## Git提交记录
1. `commit_hash` commit_message
2. ...

## 明日计划（如有）
1. ...
```

### 3. 保存日志文件
// turbo
- 将日志保存到 `D:\AISpace\EShangPython\workLog\YYYY-MM-DD.md`
- 文件名格式：`YYYY-MM-DD.md`（如 `2026-03-07.md`）
- 如果文件已存在，追加内容而不是覆盖

### 4. 提交日志
// turbo
- `git add workLog/YYYY-MM-DD.md`
- `git commit -m "docs: 更新工作日志 YYYY-MM-DD"`

## 注意事项
- **不要等用户要求才写日志**，每次会话结束都应该主动执行
- 日志语言用**中文**
- 保持简洁，重点记录：做了什么、改了什么、数据变化、发现了什么问题
- 如果会话中只做了少量工作（如仅回答问题），可以简化日志
