## ⚠️ 踩坑记录

### 坑 1: 大文件替换残留代码
- **问题**：`multi_replace_file_content` 对 7000+ 行大文件操作时，替换可能只匹配到部分目标内容，导致旧代码残留在新代码之后
- **解决**：替换后必须 `py_compile` 编译验证 + 人工检查上下文。残留代码用 Python 脚本精确删除行：
  ```python
  python -c "import pathlib; f=pathlib.Path('file'); lines=f.read_text('utf-8').splitlines(True); new=lines[:189]+lines[363:]; f.write_text(''.join(new),'utf-8')"
  ```

### 坑 2: GetTransactionTimeAnalysis `):` 丢失
- **问题**：批量替换时函数签名的 `):` 闭合括号被吞掉，导致语法错误
- **解决**：替换后 IDE lint 报错 `Expected ')'`，手动补 `):` 行

### 坑 3: Service 函数不全
- **问题**：`GetSummaryRevenueMonth`（277 行，最复杂路由）没有对应的 Service 函数
- **解决**：跳过，不强行迁移。只迁移已有 Service 函数的路由，保证安全
