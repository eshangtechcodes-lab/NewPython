## 今日修复

### 🔥 GetMonthINCAnalysis 前端504问题 → 已修复

**现象**: 前端实际调用 `GetMonthINCAnalysis?solidType=true&BusinessTradeType=1&warningType=1&calcYOY=true` 时返回504 Gateway Timeout或空数据。

**根因分析** (3个问题):

1. **Python服务已崩溃**: 进程存在但返回503，需重启
2. **数据源错误**: Python查的是 `T_REVENUEMONTHLY` 表，C#查的是 `T_BUSINESSWARNING` 表（固化模式）
3. **BusinessTradeType参数映射错误**: Python直接写 `BUSINESS_TYPE IN (1)`，C#的映射规则完全不同

**C# BusinessTradeType 映射规则（重要，其他接口也可能用到）**:
```
BusinessTradeType=1  → BUSINESS_TYPE = 4000 AND SHOPTRADE = 2    (自营餐饮)
BusinessTradeType=2  → BUSINESS_TYPE = 4000 AND SHOPTRADE <> 2   (自营非餐饮)
BusinessTradeType=3  → BUSINESS_TYPE <> 4000                     (非自营)
BusinessTradeType=1,2 → BUSINESS_TYPE = 4000                     (所有自营)
BusinessTradeType=1,3 → (BT=4000 AND ST=2) OR BT<>4000          (自营餐饮+非自营)
BusinessTradeType=2,3 → (BT=4000 AND ST<>2) OR BT<>4000         (自营非餐饮+非自营)
```

**修复内容** (`revenue_router.py` GetMonthINCAnalysis):
- 新增固化数据路径: `solidType=true` 且单月历史数据时，查 `T_BUSINESSWARNING` 表
- 实现 `_btt_where()` 函数: 6种BusinessTradeType组合的SQL条件映射
- 增加warningType→CompareField映射(WARING_VUSPD/VUSHD/VURU/VDRD)
- 生成"合计"汇总节点 (`ServerpartName="合计"`)
- 空数据时 increaseData 返回 None（对齐C#）
- 保留实时数据路径（else分支），同样修正BusinessTradeType映射

**验证结果**: 新旧API对比全部字段 ✅ 一致

### 关键发现和经验（重要）
1. **BusinessTradeType映射**: C#中1/2/3不是直接的字段值，而是映射为 `BUSINESS_TYPE=4000+SHOPTRADE=2` 等组合条件
2. **solidType路径分流**: `solidType=true`且查历史月份时，走`T_BUSINESSWARNING`固化表；否则走`T_REVENUEMONTHLY`或`T_MONTHLYREVENUE`实时表
3. **数据表差异**: CommercialApi用`T_REVENUEMONTHLY`，EShangApiMain用`PLATFORM_DASHBOARD.T_MONTHLYREVENUE`，字段名和逻辑不同
4. **WM_CONCAT**: Oracle的WM_CONCAT在DM中用`LISTAGG`替代，排序可能不同（可接受差异）
5. **日期格式**: C#使用`yyyy/M/d`（无前导零），Python需用`dt.strftime`手动格式化

### 关键技术发现
1. **T_FIELDENUM 查询**: 直接用 `FIELD_NAME = 'xxx'` 查不到数据（达梦无此字段），需用 `JOIN T_FIELDEXPLAIN ON FIELDEXPLAIN_ID WHERE FIELDEXPLAIN_FIELD = 'xxx'`
2. **T_RTSERVERPART 坐标精度**: `T_SERVERPART` 的 `SERVERPART_X/Y` 精度低（如117.0），`T_RTSERVERPART` 精度高（如116.893886），应始终优先取RT表值
3. **bool 字段**: C# Model 中 `HASCHARGE` 等用 `bool` 类型，SQL 返回的 `SUM(NVL(...))` 是 int，需 `> 0` 转 bool
4. **空商户过滤**: SQL `GROUP BY MERCHANTS_ID` 会产生 `MERCHANTS_ID=NULL` 的分组，C# Model 自动忽略，Python需手动过滤
